# 以英国为例，中西方数字生态比较研究

## 历史演进&路径依赖

**英国：** 早期对个人电脑、开放互联网标准（HTTP、DNS）和信用卡网络的广泛采用，使其走上了一条依赖“浏览器-网站-银行卡支付”模式的路径。所有后续的创新（如电商平台、SEO、数字钱包）都是作为现有基础设施的*附加层*而构建的，从而不断强化而非取代它。

**中国：** 由于缺乏上述基础设施，中国走上了一条依赖移动网络、智能手机和集成式二维码支付的不同路径。所有后续的创新（如超级应用、小程序、社交电商）都是在这个移动原生的基础上构建的，导致了完全不同的技术轨迹，并最终被“锁定”在超级应用模式中。

## 开放网络 VS 围墙花园

### A 客户从哪来

- 英国：在寻找新产品或服务时，42%的成年人会用搜索引擎，20%的社交媒体，37%经由朋友推荐^[2]^。

  (1) 企业需要客户；

  (2) 在英国，获取客户的主要渠道是谷歌 ；

  (3) 谷歌的核心功能是索引和排名网站；

  (4) 因此，为了被发现，企业必须拥有一个可以被索引和排名的网站；

  (5) 这种必要性驱动了企业在网站和SEO上的投资 ，这反过来又强化了网站作为商业信息中心的角色，进一步巩固了谷歌的门户地位。

  **相对线性的“营销漏斗” ：**^[1]^

  - **认知（Awareness）**阶段通常通过SEO或付费搜索广告产生。
  - **考虑（Consideration）**阶段则涉及在谷歌、Yelp或Trustpilot上阅读评论，并在不同企业的网站上比较信息 。在谷歌和谷歌地图之后，企业自己的网站是第三大最受信任的信息来源 。
  - **转化（Conversion）**通常发生在网站的电商平台，或是用户通过网站上的联系信息前往实体店完成购买 。

- 中国：基于“发现”而非“搜索”

  **认知**（在抖音视频中看到产品）-> **建立信任**（关注品牌微信公众号、加入社群）-> **转化**（直接通过小程序或直播购买），并且常常伴随着线下的跟进 。

| 属性             | 西方 (美国/英国)                | 中国                                  |
| ---------------- | ------------------------------- | ------------------------------------- |
| **主要商业载体** | 官方网站                        | 微信小程序 / 抖音店铺                 |
| **主要发现渠道** | 谷歌搜索与评论                  | 社交/内容信息流 (抖音, 小红书)        |
| **电商模式**     | 以网站为中心 / 平台市场         | 社交电商 / 应用内闭环                 |
| **客户旅程**     | 线性漏斗 (认知 -> 考虑 -> 转化) | 非线性生态系统 (发现 -> 互动 -> 信任) |
| **关键成功指标** | 网站流量 / 转化率               | 私域流量池规模 / 互动率               |
| **主导消费行为** | 意图驱动 (主动搜索)             | 发现驱动 / 关系导向 (被动发现)        |

### B 产品怎么卖

英国是世界第三大电子商务市场^[3]^ ，网站普及率很高。超过三分之二（68%）的英国企业拥有网站^[4]^，而在小企业中，这一比例更高，达到78%^[5]^。

电子商务平台市场主要由WooCommerce和Shopify主导，这些平台专为帮助企业建立和运营独立网站而设计，提供了品牌建设、销售和物流管理的全套工具。

### C 支付

英国的支付市场是多样化的，由借记卡/信用卡（占57%的交易）、PayPal和日益增长的移动钱包（如Apple Pay）共同主导^[6]^。

中国则是无处不在的微信支付和支付宝支付。

### D 消费者的信任差异

英国消费者对社交媒体平台上的信息表现出显著的不信任。只有不到四分之一（24%）的人信任社交媒体，大多数人认为这些平台监管不力且滥用数据^[7]^。相反，对传统媒体的信任度已回升至61%^[7]^。一个专业的网站，作为品牌自有的媒体资产，更符合这种受信任的范式。因此，企业利用网站展示客户评价、案例研究和安全认证等信任信号，对于建立信誉至关重要^[8]^。

## 在不同地域应用《做对产品》

### 不变的核心：数据、假设和“切身利益”

《做对产品》中那些与平台无关的核心原则。其核心思想包括：

- **市场失败定律（The Law of Market Failure）**：大多数新想法，即使执行得再好，也终将失败。因此，创业者面临的首要风险是“做错了产品”（building the Wrong It）。
- **数据胜于观点（Data Beats Opinions）**：你必须逃离“思想国度”（Thoughtland，即观点和调研的领域），去获取真实的用户行为数据。
- **YODA原则（Your Own DAta）**：唯一重要的数据，是你自己从目标市场收集到的、关于他们与你的想法互动的数据。
- **“切身利益”原则（Skin in the Game）**：最有价值的数据，来自于用户投入了有价值的东西（如时间、联系方式、金钱、声誉），而不仅仅是发表了一个观点。

这里的核心论点是：这些原则是普适的。创业者的目标永远是通过获取带有“切身利益”的YODA来验证一个**市场参与度假设（Market Engagement Hypothesis, MEH）**。唯一改变的是用来进行这个实验的**工具**。

### 在西方进行“预原型”测试：着陆页是终极试金石

西方的数字生态系统非常适合经典的“预原型”（Pretotyping）技术。其中，“伪装门”（Fake Door）预原型是典型的例子。

**流程如下：**

1. **构建假设**：明确提出一个可量化的假设，例如：“X%的目标受众在看到我的价值主张后，会做出Y行为（如点击‘立即购买’）”。
2. **制作工具**：创建一个简单的着陆页（即“伪装门”），清晰地阐述产品价值，并设置一个强有力的行动号召（CTA），如“预购价$50”或“加入等待列表”。
3. **导入流量**：使用精准定位的社交媒体广告（如Meta、TikTok）或谷歌搜索广告，将目标受众引流至该页面。
4. **衡量结果**：通过追踪CTA的转化率来收集YODA。当一个用户输入了他的电子邮件地址甚至信用卡信息（即使并未扣款），他就投入了显著的“切身利益”。这个数据可以直接验证或证伪最初的假设。

### 在中国进行“预原型”测试：为超级应用时代进行验证工具的本土化

在中国，一个位于开放互联网上的着陆页并非最有效的测试工具。创业者必须使用超级应用生态系统内的原生工具来开展实验。

- **微信小程序作为“伪装门”预原型**：创业者可以开发一个极其简单的单页微信小程序，替代传统的着陆页。小程序的开发成本相对低廉且速度快。这个小程序可以用来描述产品，并设置一个“上线时通知我”或“加入创始会员群”的按钮。通过二维码或在聊天中分享，它可以在目标环境中被快速传播和测试。
- **利用抖音直播和微型KOL获取实时市场反馈**：一个获取YODA的强大方式是与抖音上的一个小型、垂直领域的KOL合作。他们可以在直播中介绍你的产品概念。直播中观众的实时评论（“这个我肯定买！”、“多少钱？”）以及更重要的——点击直播间内嵌的占位产品链接的用户数量——都提供了即时的、高质量的、带有明确用户意图的市场反馈。
- **通过微信群建立测试社群**：甚至在开发小程序之前，创业者就可以围绕其产品旨在解决的问题创建一个微信群。通过在群内分享产品模型图（类似于杰夫·霍金斯的木块模型，即“匹诺曹”预原型）并观察群成员的反应和互动程度，可以收集到定性的YODA。邀请群成员预付少量定金以锁定“创始会员”折扣，则提供了一个带有“切身利益”的量化指标。

验证的“单位”也因此不同。在西方，你通常是在一个交易场景中验证一个**产品的价值主张**。西方的着陆页测试直接地问：“人们会买这个特定的东西吗？”这是对产品-市场匹配度的直接检验。而在中国，你常常是在验证一个

**社群对某个话题或解决方案的兴趣**，而产品只是该兴趣最终的实现形式。中国的微信群测试首先问：“人们是否对这个问题足够有热情，以至于愿意加入一个社群来讨论它？”这是对问题-社群匹配度的优先检验。产品随后可以与这个社群“共创”，这种模式在中国私域流量的运营中是原生行为。这意味着，中国的模式在验证早期就能提供更丰富、更定性的反馈，可能导向一个从第一天起就与社群需求更深度契合的产品。

此外，测试-学习循环的速度在中国生态系统中可能更快。虽然西方的着陆页测试已经很快，但中国的实时互动工具能够以规模化的方式提供即时反馈。一个着陆页测试需要时间来导入流量并收集到有统计学意义的数据。而一场与KOL合作的抖音直播，可能在一小时内就产生数千个数据点（评论、点击、关注）。这种反馈是即时的、对话式的，允许创业者实时提出追问。对于某些消费品而言，中国生态系统为快速、迭代的想法验证提供了无与伦比的工具，其效率甚至可能超过经典的西方预原型 playbook。

| **萨沃亚的预原型技术**             | **西方市场的实施方式**                                       | **中国市场的实施方式**                                       |
| ---------------------------------- | ------------------------------------------------------------ | ------------------------------------------------------------ |
| **伪装门 (Fake Door)**             | 简单的着陆页，带“立即购买”CTA；通过社交/搜索广告引流。       | 单页微信小程序，带“立即预订”CTA；通过在相关社群分享二维码传播。 |
| **匹诺曹 (Pinocchio)**             | 物理模型（如Palm Pilot的木块）；在Instagram上分享照片/视频。 | 在微信群中分享数字模型图/GIF，收集反馈。                     |
| **土耳其机器人 (Mechanical Turk)** | 在构建自动化软件前，通过邮件/私信手动提供服务。              | 通过微信公众号的客服消息功能手动提供服务。                   |
| **门面 (The Façade)**              | 一个“伪装门”，点击后导向一个功能简化的现有工具（如一个Typeform表单）。 | 一个小程序，用户点击后会获得一个现有类似产品的优惠券（即“贴牌”技术）。 |
| **YouTube**                        | 在着陆页上嵌入一个产品演示视频，并附上预购按钮。             | 在抖音上发布一个简短的演示视频，视频内嵌一个指向占位产品页面的链接。 |
